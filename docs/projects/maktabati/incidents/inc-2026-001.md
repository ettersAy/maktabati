## Executive Summary

**What happened:** VitePress build failed when documentation included GitHub Actions template syntax (`$\{\{ }}`) in code blocks without proper escaping.

**Why it matters:** Build errors block deployment to GitHub Pages, preventing any documentation updates from going live.

**Quick fix:** Added `\{v-pre}` directive to YAML/config code blocks containing template syntax `$\{\{ }}` to prevent Vue parser interpretation.

## Root Cause Analysis

### What Failed?

VitePress SSR renderer tried to parse `$\{\{ steps.deployment.outputs.page_url }}` as a Vue template expression, not as literal YAML syntax.

### Why Did It Fail?

By default, VitePress processes all Markdown code blocks through the Vue template compiler. When code contains `$\{\{ }}`, Vue tries to evaluate `steps.deployment.outputs.page_url` as a component property, fails to find it, and crashes.

### How Was It Triggered?

Added CI/CD documentation with real GitHub Actions workflow examples containing template literals that look like Vue expressions to the parser.

### Pattern Recognition

| Aspect | Details |
|--------|---------|
| **Root Cause** | Missing `\{v-pre}` directive on code block fence |
| **Trigger** | Adding code examples with `$\{\{` to any Markdown file |
| **Condition** | Only fails when code blocks contain template-like syntax |
| **Pattern Detection** | Error message includes undefined property names that are actually from template expressions |

## Solution Implemented

### What Was Changed?

All YAML and config code blocks with `$\{\{` or similar template syntax now include `\{v-pre}` directive.

**Example Fix:**

**Before (failed build):**
````markdown
```yaml
deploy:
  environment:
    name: github-pages
    url: \$\{\{ steps.deployment.outputs.page_url }}
```
````

**After (working):**
````markdown
```yaml {v-pre}
deploy:
  environment:
    name: github-pages
    url: \$\{\{ steps.deployment.outputs.page_url }}
```
````

**Explanation:**

The `{v-pre}` attribute prevents Vue template compilation on that code block. VitePress renders the content as-is, preserving `\$\{\{ }}` literally without attempting to evaluate it as JavaScript.

| Aspect | Value |

| **Permanent** | ✅ Yes - correct solution, not workaround |

## Key Learnings for AI Agents

**When you see this error pattern in the future:**

- **Error:** "Cannot read properties of undefined" + property name in error matches part of `\$\{\{ ... }}`
- **Cause:** Missing `{v-pre}` on code block with template syntax
- **Solution:** Add `{v-pre}` to code fence: ` ```yaml {v-pre}`
- **Remember:** This happens with: GitHub Actions, Terraform, Helm, Jinja, LaTeX, Handlebars, ERB

**What NOT to do:**
- Don't escape `$` with backslash (wrong format for VitePress)
- Don't use HTML entities (defeats purpose of showing real syntax)
- Don't remove the template syntax from examples (defeats learning purpose)

**Pattern:** Any templating language docs will need this. Check automatically when adding new docs.

## Solution

### Fix Applied

Added `v-pre` directive to code fence to skip Vue compilation:

```yaml {v-pre}
name: Deploy to GitHub Pages

jobs:
  deploy:
    environment:
      name: github-pages
      url: \$\{\{ steps.deployment.outputs.page_url \}\}
```

### Alternative Solutions

| Solution | Pros | Cons |
|----------|------|------|
| `v-pre` on code fence | Clean, minimal | Requires VitePress support |
| `<pre v-pre>` wrapper | Works universally | More verbose |
| HTML entity escape | No directives needed | Reduces readability |
