## Summary

VitePress build failed when markdown files contained GitHub Actions workflow examples with `\$\{\{ \}\}` expression syntax. Vue template compiler interpreted these as Vue bindings, causing SSR render errors.

---

## Error Details

```
✖ rendering pages...
build error:
Cannot read properties of undefined (reading 'deployment')
TypeError: Cannot read properties of undefined (reading 'deployment')
    at _sfc_ssrRender (.../guides_maktabati-setup.md.js:156:41)
```


**Root Cause:** VitePress/Vue template compiler parsed `\$\{\{ steps.deployment.outputs.page_url \}\}` as Vue template syntax instead of literal code block content.

## Solution

### Fix Applied

Added `v-pre` directive to code fence to skip Vue compilation:

```yaml {v-pre}
name: Deploy to GitHub Pages

jobs:
  deploy:
    environment:
      name: github-pages
      url: \$\{\{ steps.deployment.outputs.page_url \}\}
```

### Alternative Solutions

| Solution | Pros | Cons |
|----------|------|------|
| `v-pre` on code fence | Clean, minimal | Requires VitePress support |
| `<pre v-pre>` wrapper | Works universally | More verbose |
| HTML entity escape | No directives needed | Reduces readability |

---

## Prevention

### Documentation Standards Update

All code blocks containing template-like syntax must include `v-pre`:

| Pattern | Example | Directive |
|---------|---------|-----------|
| GitHub Actions | `\$\{\{ \}\}` | `{v-pre}` |
| Laravel Blade | `\{\{ \}\}` | `{v-pre}` |
| Vue templates | `\{\{ \}\}` | `{v-pre}` |
| Handlebars | `\{\{ \}\}` | `{v-pre}` |

### Checklist for Future Docs

- [ ] Check code blocks for template syntax patterns
- [ ] Add `v-pre` to any block with `\{\{` or `\$\{\{`
- [ ] Test build after adding new workflow examples
- [ ] Run `npm run docs:build` before committing

---

## Lessons Learned

1. **VitePress ≠ Static Markdown:** Remember VitePress processes markdown through Vue compiler
2. **Test Build Before Commit:** Always run full build, not just dev server
3. **Document Gotchas:** Add this to team knowledge base for future reference
